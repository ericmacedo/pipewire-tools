# .agents.md

This repository contains `pipewire-tools`, a CLI utility for interacting with PipeWire/WirePlumber audio sinks on Linux.

When maintaining or extending this project, adhere strictly to the following rules:

## 1. Environment & Scope
- **Domain:** Linux Audio, PipeWire, WirePlumber.
- **Dependencies:** Standard Bash, `wpctl`, `awk`, `sed`, `grep`. 
- **Constraint:** NEVER introduce new dependencies (e.g., `jq`, Python, Node.js) without explicit user permission. The tool must remain a POSIX-compliant(ish) portable bash script that relies entirely on standard tools pre-installed on most modern Linux distributions.

## 2. Architecture & Modularity
- The entrypoint is `pipewire-tools`. It is strictly responsible for argument parsing, displaying help menus, and delegating execution to the appropriate module.
- All core logic lives in `modules/<command>.sh`.
- Shared utility functions (e.g., interacting with `wpctl` or WirePlumber configs) MUST be placed in `modules/utils.sh`.
- If a new top-level command is requested (e.g., `volume`), create a new `modules/volume.sh` file. The entrypoint will automatically pick it up.

## 3. Data Storage & System State
- **State files are strictly forbidden.** Do not use custom text files (like `ignored_names.txt`) to track state.
- **Single Source of Truth:** All configuration and state must be queried directly from WirePlumber's native configuration files or active `wpctl` state.
- When injecting properties (like `node.cycle.ignore`), always read the existing `99-pipewire-tools-ignore.conf` file, parse it into memory, modify it, write the complete file back, and restart the `wireplumber` systemd user service.

## 4. Code Style
- Use `snake_case` for function and variable names.
- Prefer local variables (`local var_name=...`) within functions.
- Quote variables aggressively to prevent word splitting issues (`"$var"`).
- Keep output concise. Use `echo` for standard output and warnings, but limit verbosity unless an error occurs.
- When outputting tables (like `device list`), ensure columns are aligned nicely using `printf`.

## 5. PipeWire Specifics
- PipeWire exposes unplugged devices. Switching to them via `wpctl set-default` will fail silently. Looping logic (like in `cycle.sh`) MUST verify that the switch was successful by re-querying the active sink before breaking the loop.
- Extracting text from `wpctl status` can be fragile due to terminal formatting characters (`*`, spaces). Always test regex/awk parsers carefully.
